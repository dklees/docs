## Introduction

The Resecurity Action integration enriches indicators with threat intelligence data from Resecurity, a cybersecurity provider offering a unified platform for Cyber Threat Intelligence (CTI), Digital Risk Management, and Endpoint Protection.

**Available Action:**

- **Resecurity Data Breaches Enrichment** — enriches indicators with information about data breaches.

**Compatible Indicator Types:**

- Emails  
- FQDNs  

**Enriched Object Types:**

- Indicators  
- Reports  

**Note:** This action requires an active ThreatQ TDR Orchestrator (TQO) license.

---

## Prerequisites

- Active ThreatQ TDR Orchestrator (TQO) license  
- Resecurity API Key  
- ThreatQ data collection containing at least:
  - Email indicators  
  - FQDN indicators

## Installation

**Note:** These steps are the same for installation or upgrade.

1. Log into https://marketplace.threatq.com.
2. Download the action zip file.
3. Go to your ThreatQ integrations management page.
4. Click **Add New Integration**.
5. Upload the action zip file via:
   - Drag and drop  
   - **Click to Browse**

 **Note:**  ThreatQ will notify if the action already exists or if config changes will overwrite current settings.

You must still configure the integration after installation.

---

## Configuration

**Note:** ThreatQuotient does not issue API keys for external vendors.

Steps:

1. Go to Integrations Management in ThreatQ.
2. Filter by **Actions** (optional).
3. Open the Resecurity Action entry.
4. Enter configuration parameters.

### Configuration Parameters

| Parameter | Description |
|----------|-------------|
| **API Key** | Used to authenticate with Resecurity. |
| **Enable SSL Certificate Verification** | Validates SSL certificate. |
| **Disable Proxies** | Ignores ThreatQ proxy settings. |
| **Email Address Context Filter** | Select context to ingest: Username *(default)*, Company *(default)*, Country *(default)*, City, Industry, Job Title |
| **Breach Report Context Filter** | Select context: Compromised Data *(default)*, Geography *(default)*, Accounts *(default)*, Score *(default)* |
| **Objects per run** | Max objects processed per run. Default: `5000`. |

Save your changes when finished.

# Resecurity Data Breaches Enrichment

Queries Resecurity for known breaches associated with an email or domain. Results ingested as ThreatQ Reports.

### Endpoint

```
GET https://app.resecurity.com/api/breaches/index
```

### Sample Query Parameters

```json
{
  "query": "threatq.com",
  "per-page": 50
}
```

# Known Issues / Limitations

- `Accounts` and `Score` attributes are updatable, but final values may differ depending on ingestion order.  
- `Report` and `Email` descriptions are also updatable and may vary.  
- Ingestion is capped at **100 pages (5,000 indicators)** per input indicator.
